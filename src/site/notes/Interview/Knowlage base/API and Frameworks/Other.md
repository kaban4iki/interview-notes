## Django 

**Кратко:**  
Django — full-stack веб-фреймворк с: ORM, auth, admin, middleware, шаблоны.

**Типовой вопрос:**  
Почему Django считают «тяжёлым» фреймворком?

**Ответ:**  
Потому что он навязывает архитектуру и поставляется с большим количеством встроенных компонентов.  
Это ускоряет разработку, но снижает гибкость по сравнению с микрофреймворками.

## Flask

**Кратко:**  
Flask — минималистичный WSGI-фреймворк.  
Не навязывает архитектуру, почти всё — расширения. Синхронный по природе.

**Типовой вопрос:**  
Почему Flask плохо подходит для high-load и async?

**Ответ:**  
Flask работает поверх WSGI и не поддерживает async на уровне архитектуры.  
Даже при использовании async-функций они выполняются синхронно.  
Для high-load требуется:
- внешний async слой
- сложная инфраструктура
- строгая дисциплина разработчиков

Flask хорош для:
- простых сервисов
- внутренних админок
- прототипов

## Tornado

**Кратко:**  
Tornado — один из первых async web-фреймворков в Python.  
Имеет собственный event loop и архитектуру, отличную от ASGI.

**Типовой вопрос:**  
Почему Tornado почти не используют в новых проектах?

**Ответ:**  
Tornado:
- появился до asyncio
- имеет нестандартный API
- плохо интегрируется с ASGI-экосистемой

Сегодня его вытеснили:
- FastAPI
- Starlette
- aiohttp
